Setting up Notion

Connect HushAsk to Notion and turn closed public threads into permanent, searchable records in your Hush Library database — with a summary title you set each time.

How to connect

🔗 One-Click OAuth from App Home

Recommended

Open the HushAsk App Home tab in Slack. In the Configuration section (visible to admins only), click "Connect Notion". This opens a browser OAuth flow — authorize HushAsk in your Notion workspace and we'll automatically provision the Hush Library database. No tokens, no page IDs — done in under a minute.

🔑 Manual Token Entry

Fallback

If OAuth isn't available in your deployment, the Configuration section will show token and database ID fields instead. Create an internal Notion integration, share a database with it, and paste the credentials in. Steps below →

What gets created in Notion

When you first connect, HushAsk provisions a Hush Library database (🔒) in your Notion workspace with these properties:

• Name — the summary title you enter when archiving a thread
• Route — 📢 Public or 🔒 Confidential / HR (Select)
• Status — New → Answered → Archived (Select)
• Full message body as page content

💡 Sync is intentional, not automatic. Notion entries are only created when an admin closes a Public thread using "Close & Sync to Notion". This keeps your Hush Library clean and curated — not a raw firehose of every message received.

How archiving works

1

Admin closes a Public thread

When a public-route message is ready to archive, click "Close & Sync to Notion" on the triage message in your public knowledge base channel.

2

Set a summary title

An "Archive This Thread" modal opens. Enter a clear, searchable summary title for the entry (e.g. "How do I request PTO during blackout periods?"). This becomes the page name in Notion.

3

Entry created in Notion

HushAsk creates the page in your Hush Library with the title you set, the route, a status of "Archived", and the full message body as page content.

Manual token setup (fallback)

1

Create an internal Notion integration

Go to notion.so/my-integrations → New integration → type: Internal → name it "HushAsk" → select your workspace → Save. Copy the token (starts with secret_).

2

Create a Hush Library database

In Notion, create a new full-page Database. Add properties: Route (Select), Status (Select). This database will receive all archived HushAsk threads.

3

Share the database with your integration

Open the database → ••• → Add connections → find your HushAsk integration → Confirm.

4

Copy the Database ID

Open the database in a browser. The URL contains a 32-character ID. Copy just that string.

5

Paste credentials into App Home

Open HushAsk App Home → Configuration section → paste your token and database ID → Save.

Troubleshooting

"object_not_found" after connecting via OAuth — Hush Library provisioning failed. The integration may lack workspace-level create permissions. Try disconnecting and reconnecting, or fall back to manual setup.

Sync fails: "unauthorized" — The OAuth token has expired or been revoked. Click "Connect Notion" in App Home to reconnect.

Sync fails: "Could not find database" — The database wasn't shared with the integration (manual setup). Re-share it (Step 3 above).
