Setting up Notion

Connect HushAsk to Notion and every routed message becomes a permanent, searchable row in your Hush Library database — with one click.

Two ways to connect

🔗 One-Click OAuth

Recommended

The fastest path. During Step 3 of the Setup Wizard, click "Connect to Notion". You'll authorize HushAsk in your browser and we'll automatically provision the Hush Library database. No tokens, no page IDs — done in 30 seconds.


Available when: Your HushAsk admin has configured Notion OAuth credentials (NOTION_CLIENT_ID and NOTION_CLIENT_SECRET in the server's .env). Admin setup below →

🔑 Manual Token Entry

Fallback

If OAuth isn't configured, Step 3 shows token and database ID fields instead. Create an internal Notion integration, share a database with it, and paste the credentials in. Steps below →

What gets created

HushAsk provisions a Hush Library database (🔒) at the top level of your Notion workspace with these properties:

• Name — first 80 chars of the message
• Route — 📢 Public or 🔒 Confidential / HR
• Status — New → Answered → Archived
• Synced At — timestamp of sync
• Body of the message + blank "Add your answer" block

💡 Sync is manual, not automatic. A 📄 Sync to Notion button appears on every triage message. One click creates the entry — keeping your Notion clean and intentional.

Manual token setup (fallback)

1

Create an internal Notion integration

Go to notion.so/my-integrations → New integration → type: Internal → name it "HushAsk" → select your workspace → Save. Copy the token (starts with secret_).

2

Create (or choose) a Hush Library database

In Notion, create a new Database. Add properties: Route (Select), Status (Select), Synced At (Date). This will receive all HushAsk messages.

3

Share the database with your integration

Open the database → ••• → Add connections → find your HushAsk integration → Confirm.

4

Copy the Database ID

Open the database in browser. URL looks like: notion.so/...?v=xxx or ends with a 32-char ID. Copy just the 32-character string.

5

Paste into HushAsk Setup Wizard Step 3

App Home → Settings → Step 3 → paste token + database ID → Save & Finish.

Admin: enabling OAuth (server-side)

To enable One-Click OAuth for all workspace admins:

1

Create a Public Notion integration

Go to notion.so/my-integrations → New integration → type: Public. Public integrations support OAuth.

2

Set the redirect URI

In your integration settings, add redirect URI: http://YOUR_SERVER_IP:8080/notion/callback

3

Copy OAuth credentials to .env

Add to your server's .env file:
NOTION_CLIENT_ID=your_client_id
NOTION_CLIENT_SECRET=your_client_secret
Then restart with ./start.sh.

Troubleshooting

OAuth: "object_not_found" after connecting — Hush Library provisioning failed. The integration may lack workspace-level create permissions. Fall back to manual setup.

Sync fails: "unauthorized" — Token expired or revoked. Reconnect via Settings.

Sync fails: "Could not find database" — The database wasn't shared with the integration. Re-share it (Step 3 of manual setup).
